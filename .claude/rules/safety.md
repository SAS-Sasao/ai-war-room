# 機密情報取り扱いルール

## 保護対象
- docs/profile.md の内容（個人の弱点・キャリア目標）
- docs/minefield.md の内容（人物評価・社内政治）
- 具体的な社名・個人名が含まれるすべての情報

## 禁止事項
- 上記情報をコミットメッセージやPR descriptionに含めない
- 外部サービス（MCP経由含む）への送信時に個人名・社名を含めない
- decisions/ や logs/ に個人名を記載する場合はイニシャルまたはコードネームを使用

## Git管理
- profile.md と minefield.md は .gitignore で除外済み
- templates/ 配下のテンプレートのみGit管理する
- decisions/ と logs/ はGit管理するが、センシティブ情報は匿名化する

## .gitignore 対象（実体をコミットしない生データ）
- docs/profile.md
- docs/minefield.md
- docs/bugs.md
- docs/mental-care.md
- 実名・社名・年収・人物評価の「生データ」は上記ファイルに集約する

## 追跡ファイルへの記載ルール（公開リポジトリ前提）
Git追跡される全ファイル（docs/decisions/・docs/logs/・docs/knowledge/・docs/sample/・.claude/rules/thinking-os*.md 等）には、以下を **最初から匿名で** 書く：

| 対象 | 書き方 |
|------|--------|
| 自社（勤務先） | 「自社」 |
| クライアント | 「主要クライアント（小売）」等のコードネーム |
| 競合 | 「競合小売」「競合」 |
| ベンダー | 「ベンダーA」「ベンダーB」 |
| 実名 | 「操縦者」 |
| 個人の年収・市場価値 | 「●●万」と伏せる |
| 人物の固有評価 | 役割ベースのコードネームで一般化 |

- 公開情報源のURLは保持してよい（ドメインで所属が推測される点は許容済み）
- 企業財務・市場規模・競合の公開ランキング等の一般公開情報は記載可

## コミット前チェック（必須）
public リポジトリのため、一度コミットすると履歴・PR参照に残り、force push でも完全には消えない。
コミット前に必ず以下を確認する：
- 社名（自社・クライアント・競合・ベンダー）の実名が無いか
- 個人の実名・年収・市場価値の具体額が無いか
- ファイル名に社名が含まれていないか
