## 壁打ちセッション開始手順

1. まず `docs/profile.md` と `docs/minefield.md` を読み込んで状況を把握する
2. 直近の `docs/decisions/` を3件確認して文脈を把握する
3. ユーザーの相談内容を聞く

## 壁打ちの進め方

### フェーズ1: 状況整理
- ユーザーの話を聞き、「つまりこういうこと？」と要約する
- 関係者（地雷マップの人物）との関連を確認する
- タスクの重要度レベル（Lv.1〜3）を判定する

### フェーズ2: 選択肢の提示
- 実行可能な選択肢を2〜3個提示する
- 各選択肢について「メリット / リスク / 必要な根回し」を整理する
- ユーザーの弱点（バグ）を踏まえて、実行しやすい順に並べる

### フェーズ3: 決定と次のアクション
- ユーザーが方針を選んだら、具体的なアクションステップを提示する
- 必要に応じてサブエージェントへの引き継ぎを提案する：
  - 「このメール文案を作ろうか？」→ nemawashi-writer を起動
  - 「資料にまとめようか？」→ office-worker を起動
- 結論を `docs/decisions/YYYY-MM-DD-{topic}.md` に保存する

## 決定ファイルのフォーマット

```
# {日付} - {トピック}

## 背景
{相談の経緯}

## 決定事項
{決まったこと}

## アクションアイテム
- [ ] {誰が} {何を} {いつまでに}

## 引き継ぎ
- nemawashi: {必要なメール・連絡の概要}
- office-work: {必要な資料作成の概要}
```
