## 呼び出しモード

### `/profiler` または `/profiler reset` → フルヒアリング
初回セットアップ、または状況が大きく変わった時（転職・異動等）に使用。
4つの質問を順番にして、profile.md と minefield.md をゼロから生成する。

### `/profiler update` → 差分更新モード
ちょっとした変化を反映する時に使用。最も頻繁に使うモード。

### `/profiler update $ARGUMENTS` → ピンポイント更新
引数に更新内容を直接指定する。最も手軽。
例: `/profiler update 上司がAさんからBさんに変わった`

## フルヒアリングモード（reset）

1. まず `docs/profile.md` と `docs/minefield.md` が存在するか確認する
2. 存在する場合は「今のプロファイルをリセットして最初から作り直すけどいい？」と確認
3. 確認後、以下の4つの質問を1つずつ行う

### 質問1: 現在地
あなたの今の役職、部署、権限（予算、決裁権など）、持っているスキル（エクセル、プログラミングなど）、逆に「できないこと」や「権限がないこと」について、包み隠さず教えてください。

### 質問2: 目的地
あなたが日々の業務やプロジェクトで目指すリアルなゴールは何ですか？「出世したい」「波風立てず定時で帰りたい」「このタスクは60点で終わらせたい」など、正直なモチベーションを含めて教えてください。

### 質問3: 地雷マップ
あなたの職場の社内政治、厄介な人物（上司の性格、他部署の縄張り意識など）、絶対に踏んではいけない暗黙のルールなど、ドロドロした地形を教えてください。

### 質問4: 操縦者のバグ
あなた自身の怠惰さと思考の癖について教えてください。（例：完璧主義で手が止まる、嫌なタスクを先送りする、大風呂敷を広げがち、など）

### 出力
- 質問1, 2, 4の回答 → `docs/profile.md` に保存
- 質問3の回答 → `docs/minefield.md` に保存
- フォーマットは `docs/templates/` のテンプレートに従う

## 差分更新モード（update）

### 引数なしの場合（`/profiler update`）

1. `docs/profile.md` と `docs/minefield.md` を読み込む
2. 以下のセクションを1つずつ表示して確認する：

#### profile.md の確認
「現在地（役職・スキル・権限）はこうなってるけど、変わったことある？」
→ 変更あり → その部分だけ更新
→ 変更なし → 次へ

「目的地（ゴール・モチベーション）はこうだけど、今も同じ？」
→ 変更あり → その部分だけ更新
→ 変更なし → 次へ

「操縦者のバグはこう記録してるけど、新しい発見とかある？」
→ 変更あり → その部分だけ更新
→ 変更なし → 次へ

#### minefield.md の確認
「地雷マップに載ってる人物リスト：[一覧表示]」
「異動・退職した人、新しく追加すべき人はいる？」
→ 変更あり → 該当人物の追加/削除/更新
→ 変更なし → 次へ

「暗黙のルール：[一覧表示]」
「新しく気づいたルールや、もう無効なルールはある？」
→ 変更あり → 該当ルールの追加/削除/更新
→ 変更なし → 次へ

3. 最後に変更箇所のサマリーを表示して確認後、ファイルを保存

### 引数ありの場合（`/profiler update {内容}`）

1. `docs/profile.md` と `docs/minefield.md` を読み込む
2. 引数の内容を解析して、どのセクションの更新かを自動判定する：
   - 人物に関する情報 → minefield.md
   - 役職・スキル・権限に関する情報 → profile.md の現在地セクション
   - 目標に関する情報 → profile.md の目的地セクション
   - 暗黙のルールに関する情報 → minefield.md
3. 該当セクションのみを更新する
4. 変更内容を表示して確認後、保存する

使用例:
- `/profiler update 上司がAさんからBさんに変わった`
  → minefield.md の人物マップでAさんを更新/削除、Bさんを追加
  → 「Bさんについて地雷ポイントや攻略法を教えて」と追加ヒアリング

- `/profiler update チームリーダーに昇格した`
  → profile.md の現在地セクションの役職を更新
  → 「権限や決裁権に変化はある？」と追加ヒアリング

- `/profiler update 完璧主義は少し改善した。代わりに会議で発言できない弱点に気づいた`
  → profile.md のバグセクションを更新

## 更新履歴の記録

profile.md または minefield.md を更新した場合、
ファイル冒頭の「最終更新」の日付を今日の日付に変更する。

大きな変更があった場合は、docs/logs/ にその日の日報として
「プロファイル更新: {変更概要}」を自動追記する。
