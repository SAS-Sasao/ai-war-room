# 2026-04-08 - NEC入社90日間のメンタルお作法

## 背景
NEC入社（2026-06-01）を控え、年収520万→950万のギャップによる不安を解消したい。
ドメイン・技術準備は別リポジトリ（cc-sier-organization）で実施済み。
本件は「メンタル安定のための行動ルール」を整理する壁打ち。

## 決定事項

### 前提認識
- 950万はあなたの市場価値。前職が安すぎただけ
- 面接で見せた実力＝入社後に求められるライン。超人枠ではない
- 政治力・稟議力はNECで覚えるもの。最初から求められない

### お作法1: 最初の3ヶ月は修行期間
- 1ヶ月目: インプット期。成果ゼロでいい。NEC語・文化・チームの力学を吸収
- 2ヶ月目: 小出し期。会議で意見1つ、小タスクを1つ巻き取る程度
- 3ヶ月目: 信頼回収期。「こうした方がよくないですか？」を出し始める
- 呪文: 「今は修行中。成果は3ヶ月目から。」

### お作法2: 「前職では〜」を封印する
- 1ヶ月目は一切言わない。聞かれたら答える程度
- 代替フレーズ: 「NECではどうやるのが普通ですか？」「このやり方にはどういう経緯があるんですか？」
- 非効率に見えても3ヶ月は従う。理由がわかってから改善提案

### お作法3: 無能耐性バグへの保険
- 認知の切り替え: 「無能」→「この組織で生き残る別のスキルを持ってる人」
- 実害で判断: 自分の仕事に影響なし→スルー / 影響あり→感情でなく仕組みで対処
- ガス抜き先を社外に確保。社内で愚痴は絶対NG

### お作法4: 人見知りバグを仕組みで潰す
- ランチ作戦: 最初の2週間、毎日違う人と。「土地勘がないので教えてください」で誘う
- キーマン3人を最初の1週間で特定する（「この人に聞けば大体わかる」人）
- 朝の挨拶だけは全力。笑顔で「おはようございます」。コスパ最強

### お作法5: 「自分でやった方が早い」を封じる
- 「教えてもらう」側に回ることで関係性を作る
- やりたくなったら「これ、誰に聞くのが正解ですか？」と口に出す
- 1人で100点より、人を巻き込んで80点の方がNECでは評価される

### お作法7: ドキュメント品質を仕組みで担保する

不安の本質は「新しい環境で恥をかきたくない」。チェックリスト＋AI＋型の3つで致命的ミスを防ぐ。

#### 考え方の転換
- AIに頼る＝サボりではない。品質管理をAIに委任する仕組みを持ってるということ
- PMの仕事は「中身の判断」であって「罫線を揃える」ことじゃない

#### 提出前チェックリスト（毎回3分で回す）
- [ ] 誤字脱字チェック（AI or Word校正機能）
- [ ] 表記ゆれチェック（「サーバー/サーバ」「行う/行なう」等）
- [ ] ファイル名・シート名が命名規則に合ってるか
- [ ] 宛先・日付・版数が正しいか
- [ ] 印刷プレビューで崩れてないか（Excel/PPT）

#### 入社後すぐやること: NEC社内の「型」を抽出する
- 社内ドキュメントを5件もらってフォント・色・レイアウト・用語の統一ルールを把握
- 型をテンプレとして保存
- 中身を作ったらAIに「書式に合わせて整えて」と投げる運用にする

#### AI活用の使い分け
- 自分でやる: 構成・ストーリー設計 / 文章ドラフト / レビュー前の最終目視確認
- AIに任せる: 体裁・書式整え / 誤字脱字・表記ゆれチェック
- 原則:「判断」は自分、「作業」はAI。これがPMの正しいAI活用

### お作法6: 週1の自分振り返り
- 毎週金曜15分、以下を記録:
  - 今週うまくいったこと1つ
  - 今週しんどかったこと1つ
  - 来週のメンタル注意ポイント
- ai-war-roomの日報機能（/daily-log）を活用
