# 2026-04-09 - 強みの出し方・弱みの保険・入社1週間プラン

## 背景
ローソン本部社員（2016-04〜2019-09、店舗→店長→SV）の経歴を持ちつつ、NEC リテール系・コンビニ関連IT部署に入社する。この強みの活かし方と、プロファイルのバグがNEC環境で発動するリスクを整理し、入社1週間の具体的な行動プランに落とし込む。

---

## 決定事項

### 1. 強みの3段階リリース戦略

#### Phase 1: ステルスモード（入社〜1ヶ月目）
- 自己紹介の定型文: 「SIerでPMをやってました。小売の現場で働いてた時期もあります」
- ローソンの社名は自分からは出さない。聞かれたら答える
- 「小売の現場も少し」くらいのぼかし方
- この時期の目的は「素直な新人」ポジションの確保

#### Phase 2: 信頼構築期（2〜3ヶ月目）
- 上司との1on1で先に明かす: 「実はコンビニの現場で3年半ほど働いてまして」
- まず上司だけに伝える → 上司が活かせる場を判断してくれる
- 要件定義の場では社名を出さず一般化:
  - ✕「ローソンでは〇〇でした」
  - ◎「コンビニの現場だと、発注って朝と夕方で判断基準が全然違うんですよね」
- 知識ではなく「問い」として出す: 「この要件だと、SV巡回時のオペレーションに影響出ませんか？」

#### Phase 3: 武器化（4ヶ月目〜）
- ドメイン知識を積極的に出す。「業務がわかるPM」ポジションを確立
- ローソン × NEC の実証実験案件があれば、橋渡し役に名乗りを上げる

### 2. バグ × NECリテールBU — 発動シナリオと保険

| バグ | 発動シナリオ | 保険 |
|------|-------------|------|
| 完璧主義 | 要件の「間違い」に口出しすぎ。ドメイン知識があるから気づいてしまう | 致命的リスク（プロジェクト炎上レベル）だけ指摘。80点の要件にはスルー。完璧主義の発動条件を「品質」→「リスク」に書き換える |
| 自分でやる病 | NECの分業体制で越権行為。設計・開発・テスト・運用で担当が違う | 「自分でやる」→「人に正しく依頼する」に翻訳。PMの成果＝チームのアウトプット |
| 無能耐性 | プロセス担当者・技術弱い調整役にイラつく→顔に出る→敵を作る | 「調整を代行してくれてる人」と認知を書き換える。自分が一番苦手な調整を回してくれてる存在 |
| でかいことやりたい | 早すぎる提案で孤立。「CVS × AI で革命」を入社2ヶ月で出す | アイデアは上司に「相談」として渡す→上司が撃つ。入社1年目は「上司の得点源」になるのが最速 |
| 人見知り | マルチベンダー体制で他社・クライアントとの関係構築が遅れる | コンビニ話を会話フックに使う。「コンビニ詳しい新人」で関係が始まる |

### 3. 入社初日〜1週間の行動プラン

---

#### Day 1（6/1 月曜）— ミッション: 第一印象を固める

**朝（出社〜午前）**
- [ ] 朝は15分早く到着する。余裕があると笑顔が出る
- [ ] 挨拶は全員に。笑顔で「おはようございます、今日から配属の〇〇です」
- [ ] 自己紹介はステルス版で統一: 「SIerでPMをやってました。小売の現場も少し経験があります。よろしくお願いします」
- [ ] 自己紹介は30秒以内。長く話さない

**日中**
- [ ] PC・アカウント・Teams・メール等のセットアップ
- [ ] 直属上司との初回面談。ここで聞くこと:
  - 「最初の1ヶ月で期待することは何ですか？」
  - 「チームで今一番大きい案件はどれですか？」
  - 「困ったときに聞いていい人は誰ですか？」
- [ ] **キーマン候補を3人メモする**（お作法4）: 上司が名前を出した人、席が近い人、声が大きい人

**夕方**
- [ ] Day 1の振り返りメモ（5分）: 名前・顔・印象を記録
- [ ] 明日のランチに誘う相手を1人決める

---

#### Day 2（6/2 火曜）— ミッション: チームの地図を描く

- [ ] ランチ作戦 #1: キーマン候補 or 隣の席の人を誘う。「土地勘がないので教えてください」
- [ ] チームの案件一覧・体制図をもらう（なければ口頭で聞いてメモ）
- [ ] 社内ポータル・ナレッジベースの場所を確認する
- [ ] Teamsのチャンネル構成を把握。重要チャンネルに参加
- [ ] 会議に参加したら**発言ゼロでOK**。メモだけ取る

---

#### Day 3（6/3 水曜）— ミッション: NEC語を吸収する

- [ ] ランチ作戦 #2: 別の人を誘う
- [ ] 社内ドキュメントを5件入手する（お作法7）:
  - 提案書、要件定義書、議事録、WBSなど
  - フォント・色・レイアウト・用語の「型」を抽出
- [ ] BluStellarに関する社内資料があれば読む
- [ ] NEC固有の用語・略語をメモし始める（BU名、社内システム名、承認フロー名など）

---

#### Day 4（6/4 木曜）— ミッション: 案件の温度感をつかむ

- [ ] ランチ作戦 #3
- [ ] 担当案件（or 見学案件）の資料を読み込む
- [ ] 案件の関係者マップをメモ: クライアント側の窓口、他ベンダー、NEC社内の関連チーム
- [ ] わからないことリストを作る → 翌日以降に質問する順番を決める
- [ ] 上司に「今週気づいたこと」を軽く共有する（1〜2分の雑談レベル）

---

#### Day 5（6/5 金曜）— ミッション: 1週間を閉じる

- [ ] ランチ作戦 #4
- [ ] キーマン3人が確定しているか確認。まだなら来週に持ち越し
- [ ] 提出前チェックリスト（お作法7）をすぐ参照できる場所に配置
- [ ] **週次振り返り #0**（お作法6、15分）:
  - 今週うまくいったこと1つ
  - 今週しんどかったこと1つ
  - 来週のメンタル注意ポイント
  - ai-war-roomの /daily-log に記録
- [ ] 来週のランチ候補リストを作る（まだ話してない人を優先）

---

### 1週間を通じた鉄則

1. **発言は最小限、観察は最大限。** 意見を求められたら答えるが、自分からは出さない
2. **メモ魔になる。** 人の名前、案件名、NEC語、暗黙のルール — 全部メモ
3. **質問は「教えてください」形式。** 「なぜこうしてるんですか？」ではなく「このやり方にはどういう経緯があるんですか？」
4. **ローソンの話は封印。** 誰かが「コンビニって〇〇だよね」と振ってきても「そうなんですかね」で流す
5. **毎日5分の振り返りメモ。** 記憶が新鮮なうちに書く
